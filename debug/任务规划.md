## 三、插件系统详解

### 3.1 EventBus 事件

| 事件名 | 类型 | 触发位置 | 已订阅者 |
|--------|------|---------|---------|
| `article.published` | emit | `articles.py` | **newsletter** → 自动推送邮件 |
| `article.updated` | emit | `articles.py` | — |
| `article.deleted` | emit | `articles.py` | — |
| `article.content` | pipeline | `articles.py` GET detail | **code-snippets** → `[snippet:N]` 嵌入 |
| `comment.created` | emit | `comments.py` | — |
| `comment.content` | pipeline | `comments.py` | **katex-render** → LaTeX 渲染 |
| `theme.activated` | emit | 主题系统 | **fastblog-default / magazine / modern-minimal** → CSS 切换 |

## 六、竞品差距分析

> 对照 WordPress (全球 ~43% CMS 市占率)、Halo (Java 开源 CMS)、emDash CMS (现代 Headless CMS) 三项竞品

### 6.1 总体对比

| 维度 | FastBlog | WordPress | Halo | emDash CMS |
|------|----------|-----------|------|------------|
| **架构** | FastAPI + Astro (异步) | PHP + MySQL (同步) | Java + Spring | Go + React |
| **API 风格** | REST + MCP (AI) | REST | REST + GraphQL | GraphQL |
| **插件数量** | 11 | 60,000+ | 100+ | 插件市场 |
| **主题数量** | 3 | 10,000+ | 50+ | — |
| **移动端** | PWA + Capacitor 脚手架 | iOS/Android App | 移动端适配 | — |
| **国际化** | 4 语言 | 200+ 语言 | 多语言 | 多语言 |
| **社区规模** | 单人项目 | 全球最大 CMS 社区 | 中国活跃社区 | 海外社区 |
| **开源协议** | Apache 2.0 | GPLv2 | GPLv3 | 专有/开源版 |

### 6.2 核心差距（按优先级排序）

#### P0 — 直接影响竞争力的缺失

| # | 差距 | 说明 | 竞品对照 | 状态 |
|---|------|------|---------|------|
| 1 | **全站编辑 (FSE)** | 仅有可视化页面构建器，无模板/页眉/页脚/侧边栏在线编辑。主题结构通过 Astro 编译时固定，运行时不可调。 | WordPress Gutenberg FSE 可直接拖拽编辑整个站点布局；Halo 支持模板在线编辑 | ✅ 已补齐 |
| 2 | **插件/主题市场** | 11 个插件均为内置，无第三方分发/在线安装/更新机制。插件引擎完备但无 marketplace。 | WordPress 官方市场 + Composer；Halo 有应用市场；emDash 有插件目录 | ⛔ 已忽略 |
| 3 | **移动端原生 App** | V3 API 完备 + Capacitor 脚手架，iOS 平台已初始化，但未上架。 | WordPress 官方 iOS/Android 应用成熟；Halo 有移动版 | ⏬ 最低优先级 |
| 4 | **内容标准导出** | 支持 WordPress WXR **导入**，现已支持 WXR/JSON **导出**。 | WordPress WXR 是行业标准导入导出格式 | ✅ 已补齐 |

#### P1 — 产品体验差距

| # | 差距 | 说明 | 竞品对照 |
|---|------|------|---------|
| 5 | **富媒体编辑 UI** | 后端有图片裁剪/滤镜/旋转能力，但前端管理页无可视化编辑 UI，用户上传后无法在线调整。 | WordPress 媒体编辑器内置裁剪、旋转、缩放 |
| 6 | **评论反垃圾 (Akismet 集成)** | 内置 7 种策略（关键词/链接密度/频率/蜜罐等），但无第三方反垃圾 API 对接。垃圾评论拦截率不及 Akismet 的 99%+。 | WordPress Akismet 标准配置 |
| 7 | **缓存管理 UI** | CDN 配置页存在，但无缓存命中率统计、无「清空全部缓存」按钮、无预热功能。 | WordPress 有几十个缓存插件提供完整 UI |
| 8 | **内容日历/计划视图** | 定时发布支持列表视图，无日历/甘特图视图，无法拖拽调整排期。 | WordPress 有 Editorial Calendar 插件；emDash 有内容日历 |
| 9 | **邮件模板自定义 UI** | 邮件模板硬编码在 Python 中，管理员无法通过 UI 修改邮件内容/样式。 | WordPress 有邮件模板插件；Halo 支持模板变量 |

#### P2 — 生态与运营差距

| # | 差距 | 说明 | 竞品对照 |
|---|------|------|---------|
| 10 | **API 开发者门户** | 仅 Swagger UI，无 API Key 管理、无交互式 Playground、无 SDK 自动生成。 | WordPress REST API Handbook + 社区工具；emDash 有 GraphQL Playground |
| 11 | **WCAG 可访问性审计 UI** | 后端审核引擎完整（27KB），但管理员无法在后台查看/修复可访问性问题。 | WordPress 有 Accessibility Checker 插件 |
| 12 | **前端测试覆盖** | 仅 4 个 Vitest + 5 个 E2E 测试，覆盖严重不足。 | WordPress 核心有完整 PHPUnit + JS 测试套件 |
| 13 | **后端测试覆盖** | 17 个测试文件覆盖 792 个 Python 文件，核心服务几乎无测试。 | WordPress/WooCommerce 有数万测试用例 |
| 14 | **国际化覆盖** | 仅 zh-CN / en / ar / he 四种语言。 | WordPress 200+ 语言；Halo 多语言；emDash 多语言 |
| 15 | **文档站** | GitHub Wiki 已搭建但简陋，无版本化文档、无搜索。 | WordPress Codex + DevHub 两级；Halo 有官方文档站 |

### 6.3 FastBlog 已有的超越竞品的优势

| 领域 | FastBlog 优势 | 竞品状态 |
|------|-------------|---------|
| **AI 原生集成** | MCP Server + ReAct Agent + 写作助手 | WordPress 仅靠插件实现 |
| **实时协作** | Yjs CRDT + ProseMirror OT，多人协同编辑 | WordPress 无原生协作；emDash 有但付费 |
| **全异步架构** | FastAPI + asyncpg + 异步 Redis，高并发天然优势 | WordPress 同步 PHP 阻塞模型 |
| **性能基准** | 已集成 k6 6 场景测试套件 | 竞品多无基准 |
| **插件引擎设计** | EventBus (emit + pipeline) + SQLite 持久化 + 构建时前端注入 | 无 |
| **MCP 协议** | Model Context Protocol 服务端，AI 工具可直接操作 | 无竞品支持 |
| **RBAC 细粒度** | 字段级权限 + useCapability hook | WordPress 仅角色级 |
| **RTL 语言支持** | 原生支持阿拉伯文/希伯来文 | WordPress 需插件 |
